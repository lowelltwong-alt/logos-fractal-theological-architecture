# Codex Handoff: Scripture Authority and AI Retrieval Governance

## Mission

Create one small bridge file explaining why AI retrieval must preserve source type, authority tier, tradition scope, and asserted-vs-inferred status.

## Primary target

`docs/applications/ai-governance/scripture-authority-retrieval-governance-bridge.md`

## Required content

The bridge should explain:

- why retrieval is a theological governance issue;
- why source type labels matter;
- why citation alone is insufficient;
- how tradition scope should be displayed;
- how inferred graph relations should be disclosed;

## Guardrails

- create a universal tradition-agnostic source hierarchy without review;
- rewrite doctrine files;
- create graph objects before vocabulary review;
- treat AI summaries as authority-bearing sources;
- make this bridge canonical.

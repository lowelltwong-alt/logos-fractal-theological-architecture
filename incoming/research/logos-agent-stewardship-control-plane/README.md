# Logos Agent Stewardship Control Plane Research Seed

## Purpose

This packet stages the control-plane vocabulary and governance pattern needed before Logos ever uses specialist AI contributor agents or read-only steward agents.

It is not an executable runtime.

It is not a swarm implementation.

It is a planning seed that defines agent roles, permissions, approval gates, run-ledger expectations, and eval-harness ideas.

## Files

- `agent-role-map.yaml` — proposed specialist steward roles and limits.
- `permissions-and-action-boundaries.md` — explore/plan/execute mode boundaries and forbidden actions.
- `approval-gates.md` — future approval gates for high-risk changes.
- `run-ledger-and-provenance.md` — future run ledger and provenance expectations.
- `eval-harness-plan.md` — suggested evaluations before runtime activation.
- `codex-handoff.md` — future Codex handoff for promoting this into a governed reference document.

## Status

Incoming research only.

Do not auto-promote this packet into `.claude`, runtime code, validators, schemas, claim objects, graph objects, or active agent workflows.

## Core claim

The safe future path is specialist steward agents with narrow roles, not an uncontrolled agent swarm.

Agents may help search, propose, validate, and prepare review packets. They must not silently authoritatively change doctrine, canonical status, Bible text policy, validators, schemas, claim objects, graph objects, or merges.

# Roadmap

This roadmap organizes the Logos project in build order.

## Phase 1. Foundation
- establish repository purpose
- define companion relationship to AIRCA
- define naming, attribution, and license rules
- define the recursive artifact pattern

## Phase 2. Canon and Source Map
- identify core theologians and source traditions
- create canon nodes
- document why each source belongs in the shared core or optional overlay

## Phase 3. Doctrine Taxonomy
- define stable doctrine categories
- create doctrine nodes
- link doctrine nodes to source canon and interpretive tensions

## Phase 4. Ordering and Weighting
- define candidate ordering profiles
- define weighting profiles
- document which profiles are shared defaults versus optional overlays

## Phase 5. Derivation Logic
- build theology to ethics to governance to decision-framework chains
- preserve lineage and approval logic
- make derivations inspectable by humans and machines

## Phase 6. LAIRCA Layer
- formalize Logos-grounded configuration of Architect, Inform, Rank, Commit, Act
- define what remains theological, human, hybrid, or automatable

## Phase 7. Integrations and Use Cases
- crosswalk to AIRCA
- crosswalk to Christian AI ethics
- crosswalk to nonprofit, ministry, and institutional governance uses

## Phase 8. Research and Comparative Expansion
- compare traditions, theologians, and schools of thought
- test disagreements in ordering and weighting
- document implications for decision systems and governance

## Phase 9. Worked Examples
- build complete examples from doctrine to application
- include Habitat-style and other institutional examples

## Phase 10. Versioned Research Program
- maintain open questions
- track unresolved tensions
- add review and refresh cadence for living artifacts

# Frontier Signal Intelligence Roadmap

## Phase 1: docs-only control plane

- Define signal taxonomy.
- Define scoring model.
- Define routing policy.
- Define math, AI, algorithm, source-license, and security watch lanes.
- Add AI workflow route template.

## Phase 2: source registry integration

- Connect source/licensing watch to source registry candidates.
- Keep all source ingestion gated.

## Phase 3: generated watch reports

- Create generated watch-item summaries.
- Treat all generated reports as derived and unreviewed.

## Phase 4: review packets

- Convert high-signal items into staged research or governance PRs.

## Phase 5: runtime monitors

- Add runtime monitors only after schemas, validators, owner routing, and prompt/run-ledger policy exist.

# Logos Operating Framework Buildout Roadmap

## Purpose

This roadmap records the next major buildout phases for turning Logos into a governed theological operating system rather than only a framework-heavy repository.

## Phase 1. Governance spine
Completed or substantially seeded:
- continuation guide
- build-order policy
- self-building system
- human-in-the-loop AI build loop
- address system
- address migration model
- overlay model
- claim layer model
- validation and CI policy
- feedback and drift policies
- proposal and review workflow
- retrieval neighborhood model

## Phase 2. Root normalization
Current emphasis:
- normalize older branch roots to the newer metadata shell
- preserve IDs and anchors
- add addresses and AI-usage posture
- avoid silent theological rewrites during metadata normalization

See:
- `docs/governance/root-normalization-backlog.md`

## Phase 3. Validation strengthening
Current emphasis:
- schema registry
- minimal node and claim schemas
- frontmatter validation
- claim validation
- cross-reference validation
- later expansion into stronger graph- and overlay-aware validation

## Phase 4. Reference slice completion
Current emphasis:
- Genesis 1:26-28 passage slice
- anthropology doctrine node
- image of God theme node
- imago Dei, stewardship, and vocation concepts
- lexical links through `tselem` and `demut`
- translation and manuscript witness connection
- retrieval neighborhood package

## Phase 5. Application bridge
Next major application target:
- first AI-governance or institutional-application node derived from the Genesis anthropology slice

## Phase 6. Broader expansion
After the first slice and validation layers are stable:
- Genesis 3
- fall
- Babel
- more doctrine nodes
- more canon thinker nodes
- overlay-specific expansion

## Summary principle

The repository should continue building in this order:
- govern first
- normalize second
- validate third
- complete one slice fourth
- bridge to application fifth
- expand breadth sixth

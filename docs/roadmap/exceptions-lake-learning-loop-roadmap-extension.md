# Exceptions Lake Learning Loop Roadmap Extension

## Purpose

This roadmap extension places exceptions-lake development inside the repository's normal phased growth model.

## Phase alignment

### Near-term (governance and architecture)

- finalize nine-layer exception address grammar
- register expectation/exception/adaptation separation rules
- define AIRCA stage bridge for exception flow
- define pressure-vector classification guidance

### Mid-term (graph and machine-readable support)

- stabilize exception-event object template
- add adaptation-proposal and promotion-link object families if recurrence justifies it
- connect exception objects to relationship objects and claim-layer references
- publish review-state and trust-zone traversal guidance for exception objects

### Longer-term (operational learning loop)

- add governed scoring profiles for Rank stage prioritization
- add review-queue routing profiles by pressure vector and governance layer
- add promotion audit summaries for Act-stage changes
- add drift and recurrence dashboards tied to exception categories

## Integration checkpoints

An exceptions-lake expansion should update at least:

1. governance rules (`docs/governance/`)
2. roadmap guidance (`docs/roadmap/`)
3. graph support (`data/graph/schemes/`, `data/graph/templates/`)

This keeps exception learning part of core architecture rather than isolated import residue.

## Done condition for each expansion increment

- placement follows repository governance
- metadata and provenance are present
- trust-zone intent is explicit
- promotion path is append-only and lineage-preserving
- no collapse between ontology, taxonomy, exception capture, and adaptation action

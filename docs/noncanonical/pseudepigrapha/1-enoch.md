# 1 Enoch

> Warning: This source is noncanonical and should not be used as doctrinal authority in this repository. It may be included only for historical, comparative, or reception-history purposes.

## Why this source is included

1 Enoch is included because it is historically important for background, comparative theology, and reception-history awareness. It can help illuminate parts of the conceptual world surrounding some Second Temple and early interpretive contexts.

It is not included because this repository treats it as canonical authority.

## Classification summary

In this repository, 1 Enoch is treated as:
- noncanonical
- ancient and historically important
- potentially useful for background and comparison
- not suitable as a primary doctrinal authority

## Why it belongs in pseudepigrapha rather than the canon layer

1 Enoch should not be blended into the canon layer because the project's theological architecture needs to preserve authority distinctions clearly.

Its value is mostly in:
- historical context
- comparative motifs
- reception history
- documenting how some themes circulated outside the canonical text

## What it may be used for

This node may be used for:
- background context
- comparative theological analysis
- explaining reception history or thematic overlap
- boundary-setting in relation to canonical and doctrinal authority

## What it should not be used for

This node should not be used for:
- establishing doctrine by default
- overriding canonical scripture
- functioning as a hidden secondary canon
- smuggling speculative themes into the theological core without explicit warnings

## Why this matters for the project

The repository needs a disciplined way to document texts that matter historically without allowing them to contaminate the authority structure.

1 Enoch is therefore a useful test case for the boundary-source system:
- important enough to document
- not safe to flatten into scripture
- relevant enough to compare
- limited enough to classify carefully

# Hermeneutics Layer

This folder is the entry point for interpretive methods and reading approaches used in the repository.

Its purpose is to make interpretive assumptions explicit rather than hiding them behind doctrinal conclusions.

## Why this layer exists

A serious theological architecture needs to distinguish between:
- the biblical text
- the interpretive method applied to that text
- the reading produced by that method
- the doctrine and application drawn from that reading

This folder helps preserve that distinction.

## Possible future contents

As this layer grows, it may include files such as:
- `grammatical-historical.md`
- `canonical.md`
- `typological.md`
- `patristic.md`
- `reformed-confessional.md`
- `thomistic.md`
- `orthodox-patristic.md`

## Relation to the rest of the project

This layer sits across the scripture, canon, doctrine, and concept layers.

It helps explain why different traditions, thinkers, and doctrinal systems may read the same text differently.

# Generated Index and GraphRAG Roadmap

## Purpose

At large scale, the skill registry will need derived discovery surfaces.

## Canonical surfaces

- skill cards;
- route table;
- edge vocabulary;
- tag taxonomy;
- owner routing matrix;
- orchestrator subscription policy.

## Derived surfaces

- embeddings;
- duplicate candidate reports;
- overlap candidate reports;
- cluster/community summaries;
- orchestrator impact reports;
- GraphRAG summaries;
- search indexes.

## Rule

Derived surfaces are discovery aids. They are not canonical truth.

## Future stages

1. Manual registry docs.
2. Card schema.
3. Link/metadata validation.
4. Generated registry index.
5. Embedding index.
6. Duplicate/overlap candidate detector.
7. GraphRAG community summaries.
8. Orchestrator impact reports.
9. Runtime notifications after approval.

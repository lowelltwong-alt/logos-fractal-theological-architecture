# Duplicate, Merge, and Derivative Policy

## Duplicate candidate

A duplicate candidate is a possible overlap. It is not a deletion order.

## Merge candidate

A merge candidate proposes consolidation. It requires review of:

- source boundaries;
- tool permissions;
- model policy;
- theological profile boundary;
- side-effect level;
- audit requirements;
- owners;
- orchestrator subscriptions.

## Derivative skill

A derivative skill must declare:

- parent skill;
- changed capabilities;
- changed risks;
- changed tools;
- changed model policy;
- changed doctrinal/source profile;
- new review requirements.

## Supersession

Supersession does not delete old skills immediately. Deprecated skills should remain discoverable with a replacement pointer.

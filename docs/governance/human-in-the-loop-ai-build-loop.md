# Human-in-the-Loop AI Build Loop

## Purpose

This file defines how AI-assisted buildout should work inside the Logos repository while preserving human theological responsibility.

## Core principle

AI may accelerate repository growth.
Humans remain responsible for doctrinal authority, trust promotion, and final review of sensitive claims.

## The build loop

1. AI detects or is given a build target
2. AI drafts candidate nodes, claims, metadata, or links
3. automated validation checks structure
4. candidate enters a review queue
5. human reviewer approves, revises, rejects, or quarantines
6. approved objects are promoted into the governed repository
7. feedback metrics improve future proposal quality

## AI-allowed activities

AI may assist with:
- metadata completion
- address suggestions
- claim drafting
- duplicate detection
- broken-link detection
- candidate child-node suggestion
- retrieval-neighborhood packaging
- migration record drafting
- schema drafting
- proposal queue preparation

## Human-approval-required activities

Human approval is required for:
- doctrinal classification
- trust-zone promotion
- canonical vs noncanonical determination
- hermeneutic dispute handling
- major synthesis claims
- new relationship-verb introduction
- changes affecting shared-core vs overlay placement
- release of material as core-trusted repository content

## Review outcomes

Every significant proposal should receive one of the following outcomes:
- `approved`
- `approved_with_revision`
- `rejected`
- `quarantined`
- `deferred`

## AI posture rule

AI-generated objects should normally enter the repository in a proposal or draft state rather than appearing as silently accepted truth.

## Summary principle

The Logos repository should use AI to speed disciplined theological architecture, not to replace human theological judgment.

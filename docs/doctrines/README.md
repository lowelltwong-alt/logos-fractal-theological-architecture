# Doctrine Taxonomy

This section defines the stable doctrine categories for the Logos project.

## Purpose

Doctrine nodes are the reusable theological objects that sit between source canon and downstream ordering, weighting, governance, and application.

The goal is not merely to store beliefs, but to preserve a structured theological layer that can support derivation into ethics, institutional governance, and LAIRCA-style decision systems.

## Design rules

1. Keep doctrine categories stable even when interpretations differ.
2. Separate doctrine objects from ordering profiles.
3. Separate doctrine objects from weighting profiles.
4. Preserve tensions and disagreements rather than flattening them.
5. Link every major doctrine to source canon, downstream implications, and open questions.

## Starter doctrine categories

- doctrine of God
- Christology
- revelation and Scripture
- anthropology
- sin
- salvation
- church and community
- vocation and calling
- ethics and moral reasoning
- authority and obedience
- justice and mercy
- stewardship
- technology, mediation, and tools
- prudence, wisdom, and decision-making

## Standard doctrine node contents

Each doctrine node should eventually include:
- summary
- theological importance
- major source influences
- internal tensions
- governance implications
- AI and decision-system implications
- downstream derivations
- lineage metadata

Start by filling out anthropology, because it directly shapes agency, dignity, responsibility, automation boundaries, and the role of discernment in AI-assisted environments.

# Phase 5 - Coherence Engine and Tension Mapping

## Why Phase 5 exists

Phase 4 made the doctrine network visibly connected.

Phase 5 exists to make the network behaviorally meaningful by naming how views, assessments, and baselines reinforce, depend on, or oppose one another.

This is the first layer where the repository begins to show not only what the doctrinal objects are, but how movement in one object can affect surrounding objects.

## Why coherence and tension mapping matter

A doctrine network becomes more useful when it can express:

- what a view depends on
- what a view contradicts
- what an assessment strengthens
- what a judgment presupposes

Without this layer, the graph shows relationships but not structural pressure.

With this layer, the graph starts to show why one doctrinal adjustment may ripple outward into judgments, dependencies, and tensions.

## First-pass coherence relation types

### depends_on
Used when one object requires another as part of its intelligibility or doctrinal stability.

### contradicts
Used when one object stands in explicit structural opposition to another.

### strengthens
Used when one object reinforces the standing or force of another.

### presupposes
Used when one object assumes a baseline, framework, or prior commitment that must already be in place.

## What this phase makes newly visible

Phase 5 makes it possible to see that:

- some views stand only because neighboring views or baselines remain in place
- some views oppose one another structurally, not just historically
- some assessments reinforce a view rather than merely label it
- some judgments depend on a prior baseline and cannot be interpreted apart from it

This is the first step toward a real doctrinal ripple model.

## What Phase 5 does not yet do

Phase 5 does not yet provide:

- weighted coherence scores
- automated contradiction or dependency detection
- full cross-topic ripple mapping
- machine-run tension calculations across the repository

It is a first-pass coherence layer, not a finished reasoning engine.

## Definition of done for Phase 5

Phase 5 is done when:

- the Trinity pilot has at least one example each of depends_on, contradicts, strengthens, and presupposes
- those relations are represented as explicit relationship objects
- a coherence sidecar summarizes the relations in one place
- the sidecar includes a small ripple example showing what changes if a major view moves

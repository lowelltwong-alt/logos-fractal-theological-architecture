# Ontology Migration Buffer

`ontology/` is currently preserved as a migration buffer and reference-draft layer.

It is not a canonical authored publication surface.
For current authoritative authored content, use `docs/`.

## Current role

- Preserve draft/reference material while governed target homes are still being defined
- Support migration planning into canonical `docs/` paths
- Keep useful source material available without treating it as a competing publication surface

## Current classification rule

Ontology files should be treated as one of:
- `migrate` for material expected to move into governed `docs/` paths later
- `reference-only` for material kept as seed/reference support without current canonical publication status

## Important constraint

Do not retire or delete ontology material until target homes exist and migration notes are recorded in governance docs.

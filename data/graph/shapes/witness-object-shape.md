# Witness Object Shape

## Purpose

This file defines the expected validation shape for governed witness objects in the future primary-sources corpus.

It provides a human-readable reference for what a valid witness object should minimally contain.

## Core principle

A witness object should be structurally recognizable as part of the same primary-sources family wherever it appears.

A witness may be:
- a fuller manuscript witness
- a substantial continuous-text witness
- a textual-tradition witness where modeled that way

A witness object is not the same as a fragment object, even if some witnesses are fragmentary.

## Minimum expected shape

A governed witness object should normally include:
- identity
- classification
- witness metadata
- relationships
- provenance
- review state

## Minimum validation expectations

A governed witness object should normally have:
- stable ID
- stable anchor
- title
- witness kind
- language
- script where known
- date range where known
- material where known
- coverage
- authenticity confidence

## Higher-governance expectations

Additional validation should usually be required when a witness is:
- heavily relied on in reconstruction work
- used as a high-trust doctrinal or lexical anchor
- represented as a major textual-tradition witness
- tied to disputed provenance claims

## Summary principle

Witness objects need repeatable validation grammar so fuller textual witnesses and tradition-level witnesses remain structurally governable.

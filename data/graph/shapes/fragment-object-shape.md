# Fragment Object Shape

## Purpose

This file defines the expected validation shape for governed fragment objects in the future primary-sources corpus.

It provides a human-readable reference for what a valid fragment object should minimally contain.

## Core principle

A fragment object should be structurally recognizable as a partial surviving piece of a larger witness or text-bearing object.

A fragment object is not the same as a fuller witness object.

## Minimum expected shape

A governed fragment object should normally include:
- identity
- classification
- fragment metadata
- relationships
- provenance
- review state

## Minimum validation expectations

A governed fragment object should normally have:
- stable ID
- stable anchor
- title
- parent witness reference where known
- image or viewer references where available
- passage coverage estimate
- transcription status
- reconstruction confidence

## Higher-governance expectations

Additional validation should usually be required when a fragment is:
- central to a reconstruction claim
- linked to a disputed passage assignment
- represented with image-region or word-level overlay data
- associated with uncertain provenance or catalog information

## Summary principle

Fragment objects need repeatable validation grammar so partial witnesses remain usable without being confused with fuller manuscript witnesses.

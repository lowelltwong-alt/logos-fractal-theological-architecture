# Lexical Evidence Object Shape

## Purpose

This file defines the expected validation shape for governed lexical-evidence objects in the future primary-sources corpus.

It provides a human-readable reference for what a valid lexical-evidence object should minimally contain.

## Core principle

A lexical-evidence object should preserve original-language word or phrase evidence, contextual meaning notes, and comparison logic without collapsing grammar, semantics, and translation into one layer.

## Minimum expected shape

A governed lexical-evidence object should normally include:
- identity
- classification
- lexical-evidence metadata
- relationships
- provenance
- review state

## Minimum validation expectations

A governed lexical-evidence object should normally have:
- stable ID
- stable anchor
- title
- term ID
- lemma
- language
- passage occurrences
- semantic-range notes
- lexical-confidence indicator

## Higher-governance expectations

Additional validation should usually be required when lexical evidence:
- depends on disputed parsing or morphology
- depends on uncertain witness readings
- uses non-biblical comparative sources
- drives major translation or doctrinal claims

## Summary principle

Lexical-evidence objects need repeatable validation grammar so original-language meaning remains traceable, contextual, and confidence-aware.

# Translation Comparison Object Shape

## Purpose

This file defines the expected validation shape for governed translation-comparison objects in the future primary-sources corpus.

It provides a human-readable reference for what a valid translation-comparison object should minimally contain.

## Core principle

A translation-comparison object should preserve target-language rendering options, confidence, and contextual notes without pretending that one gloss always exhausts the meaning of the source-language expression.

## Minimum expected shape

A governed translation-comparison object should normally include:
- identity
- classification
- translation-comparison metadata
- relationships
- provenance
- review state

## Minimum validation expectations

A governed translation-comparison object should normally have:
- stable ID
- stable anchor
- title
- source language
- target language
- scope
- likely renderings
- confidence tiers or confidence notes
- context-sensitive notes

## Higher-governance expectations

Additional validation should usually be required when translation comparison:
- concerns doctrinally sensitive passages
- relies on difficult lexical or grammatical decisions
- introduces cultural or traditional interpretation notes
- is being used for public-facing or cross-tradition explanatory work

## Summary principle

Translation-comparison objects need repeatable validation grammar so language transfer remains transparent, nuanced, and tied to source evidence.

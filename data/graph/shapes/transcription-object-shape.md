# Transcription Object Shape

## Purpose

This file defines the expected validation shape for governed transcription objects in the future primary-sources corpus.

It provides a human-readable reference for what a valid transcription object should minimally contain.

## Core principle

A transcription object should preserve a governed text layer derived from a witness or fragment without collapsing transcription into reconstruction, lexical analysis, or translation.

## Minimum expected shape

A governed transcription object should normally include:
- identity
- classification
- transcription metadata
- relationships
- provenance
- review state

## Minimum validation expectations

A governed transcription object should normally have:
- stable ID
- stable anchor
- title
- transcription kind
- source witness or source fragment reference
- language
- text field or transcription payload
- reading confidence

## Higher-governance expectations

Additional validation should usually be required when a transcription is:
- diplomatic rather than normalized
- dependent on uncertain readings
- connected to image-region annotations
- relied on in lexical or doctrinally sensitive downstream analysis

## Summary principle

Transcription objects need repeatable validation grammar so the text-bearing layer remains distinct from later interpretation and reconstruction.

# Boundary Source Shape

## Purpose

This file defines the expected validation shape for governed boundary-source objects in the graph and concordance layer.

It provides a human-readable reference for what a valid restricted or boundary-sensitive source object should minimally contain.

## Core principle

A boundary-source object should be structurally recognizable as part of the same trust and restriction family wherever it appears.

## Minimum expected shape

A governed boundary-source object should normally include:
- identity
- classification
- boundary metadata
- relationships
- provenance
- review state

## Minimum validation expectations

A governed boundary-source object should normally have:
- stable ID
- stable anchor
- title
- canonical status
- confessional status
- historical authenticity
- doctrinal alignment
- use-case limit
- warning text where appropriate

## Higher-governance expectations

Additional validation should usually be required when a source is:
- forged
- heretical by tradition
- doctrinally contradictory
- highly disputed
- at risk of being misused as primary authority

## Summary principle

Boundary-source objects need repeatable validation grammar because boundary safety is one of the repository's most important governance tasks.

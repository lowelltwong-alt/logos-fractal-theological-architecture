# Passage Reconstruction Object Shape

## Purpose

This file defines the expected validation shape for governed passage-reconstruction objects in the future primary-sources corpus.

It provides a human-readable reference for what a valid passage reconstruction object should minimally contain.

## Core principle

A passage-reconstruction object should assemble evidence around a passage span without pretending that all witnesses, fragments, and transcriptions collapse into one certainty level.

## Minimum expected shape

A governed passage-reconstruction object should normally include:
- identity
- classification
- reconstruction metadata
- relationships
- provenance
- review state

## Minimum validation expectations

A governed passage-reconstruction object should normally have:
- stable ID
- stable anchor
- title
- passage ID or span
- linked witnesses
- linked fragments where relevant
- linked transcriptions where relevant
- diplomatic and/or normalized text layer
- confidence summary

## Higher-governance expectations

Additional validation should usually be required when a reconstruction:
- depends on sparse evidence
- resolves meaningful variants
- is used for lexical or translation-sensitive work
- is likely to support downstream doctrinal interpretation

## Summary principle

Passage-reconstruction objects need repeatable validation grammar so evidence assembly remains transparent, confidence-aware, and reusable.

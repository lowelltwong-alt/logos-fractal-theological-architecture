# Translation Witness Shape

## Purpose

This file defines the expected validation shape for governed translation-witness objects in the graph and concordance layer.

It provides a human-readable reference for what a valid translation witness object should minimally contain.

## Core principle

A translation witness object should be structurally recognizable as part of the same translation-governance family wherever it appears.

## Minimum expected shape

A governed translation witness object should normally include:
- identity
- classification
- translation metadata
- relationships
- provenance
- review state

## Minimum validation expectations

A governed translation witness object should normally have:
- stable ID
- stable anchor
- title
- translation class
- translation philosophy
- textual base quality
- provenance claim status
- doctrinal tendency
- confessional or sectarian association
- use-case limit

## Higher-governance expectations

Additional validation should usually be required when a translation is:
- disputed
- sectarian
- restorationist
- claimed to preserve special hidden antiquity
- high-risk for doctrinal manipulation

## Summary principle

Translation witness objects need repeatable validation grammar because translation is one of the repository's most sensitive trust layers.

# Doctrines Scheme

## Purpose

This file describes the doctrines domain within the graph and concordance layer.

A doctrines scheme groups machine-readable doctrine objects that function as reusable theological anchors across scripture, canon thinkers, concepts, and downstream reasoning.

## Why this scheme exists

Doctrine objects should not remain only prose notes or isolated markdown files.

A scheme gives them:
- bounded domain identity
- clearer graph governance
- easier traversal
- stronger reuse across interpretation and application

## What belongs here

Typical objects in this scheme include doctrine objects such as:
- anthropology
- creation
- sin
- grace
- covenant
- ecclesiology

## Relation to the rest of the repository

Doctrine objects in this scheme should connect strongly to:
- scripture passages
- biblical theme nodes
- concept objects
- canon thinker nodes
- relationship objects

## Summary principle

A doctrines scheme helps the repository treat doctrine as a governed graph domain rather than only as long-form prose.

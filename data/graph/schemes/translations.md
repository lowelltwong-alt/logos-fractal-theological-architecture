# Translations Scheme

## Purpose

This file describes the translations domain within the graph and concordance layer.

A translations scheme groups machine-readable translation witness objects that function as governed carriers of rendering choices, trust signals, provenance claims, and use-case limits.

## Why this scheme exists

Translation witnesses are too important to remain only incidental citations.

A scheme gives them:
- bounded domain identity
- consistent trust governance
- easier comparison across renderings
- stronger machine-readable linkage to passages, terms, and doctrinal use

## What belongs here

Typical objects in this scheme include translation witness objects such as:
- ESV
- NRSVUE
- LXX as versional witness where modeled that way
- disputed or sectarian translation objects

## Relation to the rest of the repository

Translation objects in this scheme should connect strongly to:
- scripture passages
- lexical term nodes
- manuscript witnesses
- boundary and trust logic
- relationship objects where translation-sensitive edges matter

## Summary principle

A translations scheme helps the repository treat translation witnesses as a governed graph domain rather than only as named versions in prose.

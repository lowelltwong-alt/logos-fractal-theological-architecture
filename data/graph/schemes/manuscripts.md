# Manuscripts Scheme

## Purpose

This file describes the manuscripts domain within the graph and concordance layer.

A manuscripts scheme groups machine-readable witness objects that function as governed references for textual traditions, manuscript families, and text-critical context.

## Why this scheme exists

Manuscript and witness objects should not remain only scattered explanatory notes.

A scheme gives them:
- bounded domain identity
- clearer witness governance
- easier text-critical traversal
- stronger linkage to passages, translations, and lexical interpretation

## What belongs here

Typical objects in this scheme include witness objects such as:
- MT
- LXX where modeled as witness/tradition
- Dead Sea Scrolls witness objects
- papyri and codex objects where relevant

## Relation to the rest of the repository

Witness objects in this scheme should connect strongly to:
- scripture passages
- translation objects
- lexical nodes
- text-critical notes
- provenance and confidence logic

## Summary principle

A manuscripts scheme helps the repository treat textual witnesses as a governed graph domain rather than only as background notes.

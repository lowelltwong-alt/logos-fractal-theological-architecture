# Graph Schemes

## Purpose

This folder is the entry point for graph and concordance schemes.

A scheme is a bounded domain or structured grouping that helps machines and contributors understand what family of graph objects belongs together.

## Why this layer exists

The repository is becoming large enough that graph objects should not only exist as individual files. They should also be grouped into intelligible domains.

Schemes make it easier to:
- search
- validate
- traverse
- govern
- reason about domain boundaries

## Typical scheme domains

Examples may include:
- concepts
- doctrines
- verses
- passages
- relationships
- translations
- manuscripts
- boundary sources
- exceptions lake

## Relation to the rest of the repository

This layer should be read together with:
- `docs/governance/fractal-rules.md`
- `docs/governance/competency-questions.md`
- `docs/governance/interface-patterns.md`
- `docs/governance/shared-properties.md`

## Summary principle

A strong graph is easier to govern when its objects belong to intelligible, bounded schemes rather than one undifferentiated mass.

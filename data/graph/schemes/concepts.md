# Concepts Scheme

## Purpose

This file describes the concepts domain within the graph and concordance layer.

A concepts scheme groups machine-readable concept objects that function as reusable conceptual anchors across scripture, doctrine, canon thinkers, and graph traversals.

## Why this scheme exists

Concept objects should not remain an undifferentiated collection.

A scheme gives them:
- bounded domain identity
- clearer governance
- easier traversal
- stronger validation expectations

## What belongs here

Typical objects in this scheme include reusable concept objects such as:
- creation
- covenant
- word
- vocation
- institutions
- public reason

## Relation to the rest of the repository

Concept objects in this scheme should connect strongly to:
- doctrine nodes
- canon thinker nodes
- biblical theme nodes
- relationship objects
- graph traversal and concordance use cases

## Summary principle

A concept scheme helps the repository treat concepts as a governed graph domain rather than a loose collection of labels.

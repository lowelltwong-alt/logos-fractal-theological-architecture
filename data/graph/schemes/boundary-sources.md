# Boundary Sources Scheme

## Purpose

This file describes the boundary-sources domain within the graph and concordance layer.

A boundary-sources scheme groups machine-readable objects for noncanonical, pseudepigraphal, forged, heretical, sectarian, or otherwise restricted materials that must be documented without being silently treated as core doctrinal authorities.

## Why this scheme exists

Boundary-sensitive materials need stronger structural handling than casual mention.

A scheme gives them:
- bounded domain identity
- explicit trust and warning logic
- easier comparison without flattening authority
- safer machine-readable traversal

## What belongs here

Typical objects in this scheme include:
- noncanonical text objects
- pseudepigraphal source objects
- forged or heretical source objects
- disputed or sectarian translation objects where the graph needs explicit restriction metadata

## Relation to the rest of the repository

Boundary-source objects in this scheme should connect strongly to:
- trust zones
- warning logic
- translation and noncanonical classification systems
- relationship objects where use-case limits matter
- provenance and review logic

## Summary principle

A boundary-sources scheme helps the repository document dangerous or limited materials without letting them silently contaminate the authority structure.

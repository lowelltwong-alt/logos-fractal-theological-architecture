# Relationship Objects

## Purpose

This folder holds cross-reference edge objects when a relationship becomes important enough to preserve as its own governed artifact.

A simple graph link is often enough.
But some relationships deserve their own object because they need:
- provenance
- rationale
- review state
- trust status
- later supersession or dispute tracking

## Core rule

Do not turn every small link into a separate object.

Create a relationship object when the edge itself becomes semantically or operationally important.

## Good uses

- major verse-to-verse intertextual claims
- verse-to-concept claims that may be reused often
- contested or high-impact doctrinal support claims
- translation-sensitive connections
- claims likely to need later review or supersession

## Summary principle

Important edges should be first-class where needed.
That is one of the strongest protections against hidden drift in a large concordance graph.
